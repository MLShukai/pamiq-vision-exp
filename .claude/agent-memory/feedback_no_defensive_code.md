---
name: No defensive code
description: 防御的コードを書かない。エラーになるべきところは自然にエラーを出す
type: feedback
---

エラーになるべき箇所には防御的なコード（None初期化 + 後からNoneチェック、不要なデフォルト値、冗長なバリデーション等）を書かないこと。自然にエラーが発生するようにする。

**Why:** 防御的コードは本質的でない冗長さを生み、バグを隠す原因になる。エラーは早期に明確に発生すべき。

**How to apply:**

- 必ず値が設定されるフィールドを `None` で初期化しない
- フレームワークや型システムが保証するバリデーションを手動で重複しない
- `if x is None: raise` のようなチェックより、そもそも None にならない設計を選ぶ
- ただし適切なエラーメッセージを出すためのバリデーションは正当（例: loader.pyのFPSチェック）
- ClearMLなど常に使用する依存はtry-exceptで囲まず直接importする
