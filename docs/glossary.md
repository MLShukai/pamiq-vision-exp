# 用語集

本プロジェクトで使用する用語の定義。

## 学習パラダイム

| 用語                   | 定義                                                                                                                                                                                  |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Continuous Learning    | 本プロジェクトが対象とする学習パラダイム。データがストリーミングとして逐次到着し、学習と推論が同時並行で行われる。Online Streaming Learning に近い概念。Continual Learning とは異なる |
| Continual Learning     | タスク逐次学習と破壊的忘却の回避を主眼とした学習パラダイム。本プロジェクトの対象外                                                                                                    |
| 非定常性（分布シフト） | 入力データの統計的性質が時間とともに変化する現象。一人称ロボット視点では環境の変化により常に存在する                                                                                  |

## 表現学習

| 用語                 | 定義                                                                                                          |
| -------------------- | ------------------------------------------------------------------------------------------------------------- |
| 表現学習             | 高次元の入力データ（動画フレーム）を低次元の特徴量空間に写像する手法の総称                                    |
| 特徴量空間           | エンコーダによって入力が写像される低次元の空間                                                                |
| 表現崩壊（collapse） | 特徴量が潰れて一定値に近づき、入力の情報をほとんど保持しなくなる現象                                          |
| EMA（指数移動平均）  | Target Encoder のパラメータ更新に使用する。Context Encoder のパラメータの移動平均で Target Encoder を更新する |

## V-JEPA 関連

| 用語                      | 定義                                                                                                              |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| V-JEPA                    | Video Joint Embedding Predictive Architecture。マスクされた動画パッチの潜在表現を予測することで表現を学習する手法 |
| チューブレット（tubelet） | 動画を時空間方向に分割したパッチ。3D 畳み込みにより生成される                                                     |
| Context Encoder           | マスクされたチューブレットを入力とし、可視部分から表現を学習するエンコーダ                                        |
| Target Encoder            | Context Encoder の EMA コピー。マスクなしで全チューブレットを処理し、教師信号を生成する                           |
| Predictor                 | Context Encoder の出力から Target Encoder の出力を予測するモジュール                                              |
| 3D RoPE                   | 3 次元の回転位置埋め込み。時間・高さ・幅の 3 軸に対する位置情報をエンコードする                                   |
| マスキング                | 入力チューブレットの一部を隠し、残りから予測させる学習戦略。手法の原論文に従う                                    |

## データパイプライン

| 用語                 | 定義                                                                              |
| -------------------- | --------------------------------------------------------------------------------- |
| フレームスタッキング | 連続する複数フレームを束ねて 1 つの入力サンプルを構成する処理                     |
| リプレイバッファ     | 過去のデータを蓄積し、学習時にサンプリングするためのデータ構造。デフォルトは FIFO |
| 学習トリガー         | バッファに N フレーム分のデータが蓄積された時点で学習ループを開始する条件         |
| フェード処理         | 複数動画間の遷移時に、黒画面へのフェードアウト・フェードインを行う処理            |

## 評価

| 用語             | 定義                                                                                                                 |
| ---------------- | -------------------------------------------------------------------------------------------------------------------- |
| 入力再構成       | エンコーダの特徴量から元の入力を復元する評価タスク。特徴量が入力情報を保持しているかを検証する                       |
| 未来予測         | 特徴量空間で過去の系列から未来の特徴量を予測する評価タスク。特徴量が世界のダイナミクスの近似に適しているかを検証する |
| 予測ホライズン   | 未来予測で何ステップ先まで予測するかの設定。1, 2, 4, 8 ステップと倍々に設定する                                      |
| チェックポイント | 学習中に定期的に保存されるモデルパラメータのスナップショット。評価パイプラインの入力となる                           |
| ベースライン     | 評価の比較対象。ダウンサンプリング（リサイズ）や VAE 等、エンコード結果と同バイト量の手法を用いる                    |

## ツール・フレームワーク

| 用語              | 定義                                                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------- |
| Hydra             | 設定管理フレームワーク。YAML ファイルで全設定を管理し、experiment config による上書きを可能にする |
| experiment config | Hydra の `experiment/` 配下の YAML。モデル・データ・学習設定を実験単位でまとめて上書きする        |
| ClearML           | 実験管理・ログ記録ツール                                                                          |
| minGRU            | 未来予測の評価モデルとして使用する軽量 RNN。シンプルかつ並列化可能                                |
