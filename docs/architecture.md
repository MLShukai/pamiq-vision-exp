# システムアーキテクチャ

## 全体構成

本プロジェクトは **学習パイプライン** と **評価パイプライン** の 2 つの独立したパイプラインで構成される。

```
┌─────────────────────────────────────────────────────────┐
│                    学習パイプライン                        │
│                                                         │
│  動画ファイル → フレーム抽出 → スタッキング → バッファ      │
│                                                ↓        │
│                              トリガー → 学習ループ        │
│                                          ↓              │
│                                   チェックポイント保存     │
└─────────────────────────────────────────────────────────┘
                          ↓ チェックポイント群
┌─────────────────────────────────────────────────────────┐
│                    評価パイプライン                        │
│                                                         │
│  動画ファイル → 凍結エンコーダ → 特徴量データセット         │
│                                       ↓                 │
│                    Decoder 学習 / RNN 学習                │
│                                       ↓                 │
│                              誤差計算・記録               │
└─────────────────────────────────────────────────────────┘
```

## 学習パイプライン

### コンポーネント

#### 1. データローダー

動画ファイルからフレームをシーケンシャルに読み出し、前処理・スタッキングを行う。詳細は [data.md](data.md) を参照。

#### 2. リプレイバッファ

- 置換可能なインターフェースを持つ（デフォルト: FIFO）
- ポインタベースのメモリ管理で効率化
- 学習時のサンプリングソースとして機能

#### 3. 学習ループ

```
while 動画にフレームが残っている:
    バッファにフレームを蓄積（N フレーム分）

    if トリガー条件を満たす:
        for epoch in range(num_epochs):
            for batch in バッファからサンプリング(batch_size=B):
                モデルを学習

        if チェックポイント保存タイミング:
            パラメータを保存
```

#### 4. 表現学習モデル（エンコーダ）

- 入力: スタッキングされたフレーム
- 出力: 低次元の特徴量
- 手法は置換可能（V-JEPA 等）。詳細は [models.md](models.md) を参照

#### 5. チェックポイント管理

- 現実時間スケール（秒単位）でパラメータを保存
- 各チェックポイントに動画上のタイムスタンプを紐づけ

##### 保存先

```
logs/<実験名>/<実験開始時刻>/checkpoints/<時刻>.ckpt/
```

##### 保存内容

- **モデルパラメータ**: `torch.save` で保存
- **リプレイバッファ**: 動画データを圧縮して保存（形式は未決）

### 設定項目

| 項目                 | 説明                             | 単位               |
| -------------------- | -------------------------------- | ------------------ |
| トリガー間隔         | 学習を開始するデータ蓄積量       | フレーム数 or 秒数 |
| バッチサイズ         | 1 回の勾配更新あたりのサンプル数 | -                  |
| エポック数           | トリガーあたりの学習エポック数   | -                  |
| チェックポイント間隔 | パラメータ保存の間隔             | 秒                 |
| バッファサイズ       | リプレイバッファの上限           | フレーム数         |

## 評価パイプライン

学習パイプラインとは完全に分離されたオフラインプロセス。詳細は [evaluation.md](evaluation.md) を参照。

### コンポーネント

#### 1. 特徴量生成

- チェックポイントからエンコーダパラメータを復元し凍結
- 動画全体のフレームをエンコードし、特徴量データセットを生成

#### 2. 入力再構成評価

- 軽量 CNN Decoder（ConvTranspose）を特徴量データセットに対して学習
- ピクセル空間での再構成誤差（MAE, MSE）を全フレームに対して計算

#### 3. 未来予測評価

- 軽量 RNN（minGRU 等）を特徴量系列に対して学習
- 特徴量空間での予測誤差を計算（予測ホライズン: 1, 2, 4, 8 ステップ）

#### 4. ベースライン評価

- ダウンサンプリング・VAE 等のベースラインに対しても同一の評価を実行
- 同バイト量で公平に比較

## モジュール構成

```
src/exp/
├── data/              # データパイプライン
│   ├── loader.py      # 動画フレームローダー
│   ├── stacking.py    # フレームスタッキング
│   └── buffer.py      # リプレイバッファ（置換可能）
├── models/            # 表現学習モデル
│   ├── vjepa/         # V-JEPA
│   └── ...            # 他の手法
├── training/          # 学習ループ
│   ├── loop.py        # メイン学習ループ
│   └── checkpoint.py  # チェックポイント管理
├── evaluation/        # 評価パイプライン
│   ├── reconstruction.py  # 入力再構成評価
│   ├── prediction.py      # 未来予測評価
│   └── baseline.py        # ベースライン
├── train.py           # 学習エントリーポイント
└── eval.py            # 評価エントリーポイント
```

## エントリーポイント

| コマンド   | Hydra 設定           | 役割                                     |
| ---------- | -------------------- | ---------------------------------------- |
| `train.py` | `configs/train.yaml` | 動画からの表現学習とチェックポイント保存 |
| `eval.py`  | `configs/eval.yaml`  | 入力再構成・未来予測の評価を一括実行     |

`eval.py` は入力再構成と未来予測の両方をまとめて実行する。必要に応じて個別に実行可能な設計とする。

## 設定管理

Hydra を使用し、全設定を YAML ファイルで管理する。[lightning-hydra-template](https://github.com/ashleve/lightning-hydra-template) のパターンを参考にし、experiment config による実験単位の設定上書きを可能にする。

```
configs/
├── train.yaml         # 学習用メイン設定
├── eval.yaml          # 評価用メイン設定
├── model/             # モデル別設定
│   ├── vjepa.yaml
│   └── ...
├── data/              # データ設定
├── training/          # 学習ループ設定
├── evaluation/        # 評価設定
└── experiment/        # 実験設定（各設定を上書き）
    ├── vjepa_short.yaml
    └── ...
```

### experiment config

`experiment/` 配下の YAML で、モデル・データ・学習設定を実験単位でまとめて上書きできる。個別の設定ファイルを変更せずに実験のバリエーションを管理する。

```bash
# 例: experiment config を指定して学習
python train.py experiment=vjepa_short
```

## 既存コードの扱い

現在の `src/exp/` にある PAMIQ Core ベースの実装（trainers, agents 等）は使用せず、クリーンに作り直す。PAMIQ Core は依存関係から除外する。

## 実験管理・ログ

**ClearML** を使用する（AIM は Python 3.13 非対応のため除外）。
