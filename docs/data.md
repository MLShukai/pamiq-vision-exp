# データパイプライン

## 入力データ

- **形式**: 事前に記録された一人称ロボット視点の動画ファイル
- **性質**: 時間的に連続したフレーム列。非定常性（分布シフト）を常に含む

## パイプライン概要

```
動画ファイル → フレーム抽出 → フレームスタッキング → リプレイバッファ → 学習トリガー → 学習ループ
```

### 1. フレーム抽出

動画ファイルからフレームを順次読み出す。ストリーミング的にシーケンシャルに処理し、ランダムアクセスは行わない。

#### フレームレート

- 全ての動画で**一定のフレームレート**として扱う（デフォルト: **10 FPS**）
- 元の動画が高 FPS の場合はサブサンプリングする（例: 30 FPS の動画 → 3 フレームごとに取得）
- FPS は設定ファイルで指定可能

#### 前処理

1. **クロップ**: ターゲットアスペクト比に合うよう中央領域をクロップ
2. **リサイズ**: 指定サイズにリサイズ
3. **標準化**: ピクセル値の正規化

#### 複数動画の扱い

複数の動画ファイルを順次読み込む。動画間の遷移時は以下のフェード処理を行う:

- 現在の動画 → **黒画面へフェードアウト**（デフォルト: 1 秒）
- 黒画面 → **次の動画へフェードイン**（デフォルト: 1 秒）
- フェード時間は設定ファイルで調整可能

### 2. フレームスタッキング

連続する複数フレームをスタッキングして 1 つの入力サンプルを構成する。

| パラメータ   | 説明                   | デフォルト値  |
| ------------ | ---------------------- | ------------- |
| `num_frames` | スタッキング枚数       | V-JEPA と同じ |
| `stride`     | フレーム間のストライド | `1`           |

### 3. リプレイバッファ

スタッキングされたフレームを蓄積し、学習データとして供給する。

#### 基本方針

- デフォルトは **FIFO のシーケンシャルバッファ**
- バッファ戦略は**置換可能な設計**とする（バッファの種類が性能に大きく影響する可能性があるため）
- 全データ保持は現実的に不可能なため、古いデータは破棄する
- 画像データはポインタ的に管理し、メモリ効率を考慮する

#### バッファの要件

- FIFO を基本とし、他のサンプリング戦略も実装可能なインターフェースを持つ
- サイズ上限を設定可能
- 画像データの実体コピーを最小限に抑える効率的なデータ管理

### 4. 学習トリガーとループ

学習のトリガーとバッチ学習は独立した概念として扱う。

- **トリガー条件**: N フレーム分のデータがバッファに蓄積されたら学習を開始する
  - N 秒で指定した場合は、動画の FPS 設定値からフレーム数に換算する
- **学習ループ**: トリガー後、バッチサイズ B × エポック数で学習を実行する（デフォルトエポック数: **1**）
- トリガー条件とバッチサイズは非依存であり、それぞれ設定ファイルで調整可能

```
[データ蓄積: Nフレーム] → [学習: B × epochs] → [データ蓄積: Nフレーム] → [学習: B × epochs] → …
```
