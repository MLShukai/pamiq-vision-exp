# プロジェクト概要

## 目的

本プロジェクトは、Continuous Learning（連続学習）における Vision モジュールの表現学習手法を実験・評価するためのリポジトリである。

具体的には、一人称ロボット視点のストリーミング動画から得られる視覚情報を低次元の特徴量空間に写像する表現学習モデルを、連続学習の環境下で訓練し、その性能を比較検証する。

## 背景

### Continuous Learning とは

本プロジェクトが対象とする Continuous Learning は、一般的な Continual Learning（タスク逐次学習・破壊的忘却の回避が主眼）とは異なる。

本プロジェクトにおける Continuous Learning は、Online Streaming Learning に近い概念であり、以下の特徴を持つ：

- データがストリーミングとして逐次到着する
- 学習と推論が同時並行で行われる
- 事前に定義されたタスク境界を持たない
- データの非定常性（分布シフト）が常に存在する
- 現実世界での活動を念頭に置く

### 表現学習の位置づけ

エージェントが環境から受け取る高次元の視覚情報（一人称視点の動画フレーム）を、下流タスクで利用可能な低次元の特徴量に変換する役割を担う。本プロジェクトでは、この表現学習部分に焦点を当て、様々な手法の適用可能性と性能を評価する。

## プロジェクトの目標

学習された表現の品質を、以下の 2 つの観点から評価する。これらは必ず**両方**を行う必要がある。

### 1. 入力再構成（情報保持の検証）

特徴量が入力情報を十分に保持しているかを確認する。入力再構成ができない場合、表現空間が崩壊（collapse）している可能性がある。

### 2. 未来予測（ダイナミクス近似の検証）

低次元に写像された特徴量空間で RNN 等を用いて未来予測タスクを行い、予測誤差によって表現の質を判断する。世界のダイナミクスをどれだけ近似しやすい「良い」特徴量が得られているかを測る。

### なぜ両方が必要か

| 状態           | 入力再構成 | 未来予測         | 解釈                                                         |
| -------------- | ---------- | ---------------- | ------------------------------------------------------------ |
| 表現崩壊       | 不可       | 容易（見かけ上） | 特徴量が潰れているだけで有意義な予測ではない                 |
| 過剰な情報保持 | 良好       | 困難             | 入力を完璧に保持するが抽象化されておらず、予測タスクに不向き |
| 良い表現       | 良好       | 良好             | 入力情報を保持しつつ、ダイナミクスの近似にも適した抽象化     |

## スコープ

### 対象範囲

- 一人称ロボット視点のストリーミング動画からの表現学習
- 複数の表現学習手法の実装と比較（V-JEPA 等）
- Continuous Learning 環境下での性能評価（入力再構成・未来予測）

### 対象外

- 明示的な「タスク」の概念（ピッキング、移動等）
- 非連続なデータ（ランダムな画像集合からの学習）
- 表現学習手法そのものの新規提案
- 下流タスク（制御・意思決定等）の実装

## 前提条件

- 入力データの時間的一貫性は担保されない。一貫性の変動ペースを把握することも目的の一つである
- データの非定常性は常に存在し、それ自体を排除する対象ではない

## 技術基盤

- **学習ループ**: リプレイバッファにデータを蓄積し、一定量溜まったら学習を実行する強化学習的ループ
- **入力データ**: 事前に記録された一人称ロボット視点の動画ファイル
- **入力形式**: 動画フレームをスタッキングした時系列画像データ
- **言語・ツール**: Python 3.13+, PyTorch, Hydra

### PAMIQ Core を使用しない理由

PAMIQ Core はリアルタイム環境での学習と推論の同時実行を主眼としたフレームワークである。本プロジェクトでは入力データが事前に記録済みの動画であり、リアルタイム性の制約がないため、よりシンプルな学習パイプラインで十分である。PAMIQ Core は依存関係から除外する。
