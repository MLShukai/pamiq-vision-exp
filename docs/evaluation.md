# 評価手法

## 概要

表現学習モデルの品質を、**入力再構成**と**未来予測**の 2 軸で評価する。単一の指標では表現の崩壊や過剰な情報保持を見抜けないため、両方を組み合わせて総合的に判断する。

## 評価パイプライン

評価はすべて**オフライン**で行う。学習中にオンラインで評価モデルの学習は行わない。

### 手順

1. **エンコーダの学習**: 動画ストリーム上で表現学習モデルを学習し、定期的にチェックポイントパラメータを保存する（現実時間スケールの秒単位で設定可能。初期実験では 5 分間隔、スケールアップ時は 1 時間間隔を想定）
2. **特徴量の生成**: 各チェックポイントのエンコーダパラメータを凍結し、動画全体のフレームをエンコードする
3. **評価モデルの学習**: 凍結された特徴量に対して Decoder・RNN を学習する（静的データセットとして扱う）
4. **誤差の記録**: 全フレームに対してデータポイントごとの誤差を計算し、`.pt` ファイルとしてチェックポイントディレクトリ内に保存する。集約メトリクス（平均値）とデータポイントごとの誤差テンソルの両方を記録する

## 評価軸

### 1. 入力再構成

特徴量が入力情報を十分に保持しているかを検証する。

- **仮説**: エンコーダは入力情報を保持するはず。すなわち、ある程度は復元できる
- **方法**: **軽量な** CNN ベースの Decoder（ConvTranspose）を学習し、ピクセル空間で再構成する
- **指標**: 絶対誤差（MAE）、二乗誤差（MSE）
- **軽量である理由**:
  - 高性能な Decoder では Decoder 自体の能力で誤差が小さくなり、表現の品質を正しく測れない
  - Decoder の学習コストを抑える

### 2. 未来予測

特徴量空間が世界のダイナミクスの近似に適しているかを検証する。

- **仮説**: 抽象化された「良い」特徴量ならば、未来予測タスクが解きやすくなっているはず
- **方法**: 軽量な RNN でエンコードされた特徴量の遷移を学習し、未来の特徴量を予測する
- **予測モデル**: minGRU 等のシンプルかつ並列化可能な RNN を使用する。Transformer のようにコンテキスト長に比例して計算量が増大するモデルは使用しない。候補: minGRU, Mamba, RWKV, 純粋な RNN
- **予測ホライズン**: 設定ファイルで調整可能とする。1, 2, 4, 8 ステップ先と倍々に設定する
- **予測空間**: 特徴量空間のみ（ピクセル空間での未来フレーム予測は行わない）
- **指標**: 特徴量空間での絶対誤差等

### 評価の組み合わせ解釈

| 入力再構成 | 未来予測 | 解釈                                                        |
| ---------- | -------- | ----------------------------------------------------------- |
| 不良       | 良好     | 表現崩壊の疑い — 潰れた特徴量で見かけ上の予測精度が出ている |
| 良好       | 不良     | 過剰な情報保持 — 抽象化が不足し予測に不向き                 |
| 良好       | 良好     | 良い表現 — 情報保持と抽象化のバランスが取れている           |
| 不良       | 不良     | 学習不足または手法の不適合                                  |

## ベースライン

評価の比較対象として、エンコード結果と**同じバイト量**になる手法を用いる。

| ベースライン       | 説明                                                               |
| ------------------ | ------------------------------------------------------------------ |
| ダウンサンプリング | 画像のリサイズ関数による単純な圧縮                                 |
| VAE                | 変分オートエンコーダによる潜在表現（同条件で Continuous Learning） |

すべてのベースラインは、評価対象の表現学習モデルと**同じストリーミングデータ・同じ Continuous Learning 条件**で学習する。

## 結果の分析

- 各チェックポイントにおけるデータポイントごとの誤差の推移をプロットする（error vs data point index）
- 指数移動平均（EMA）による平滑化を適用し、過去→未来方向の誤差傾向を可視化する
- チェックポイント間の平均誤差の変化を比較し、学習の進行に伴う表現品質の変動を観察する
- ベースラインとの差分により、表現学習手法の付加価値を定量化する
